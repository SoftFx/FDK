SoftFX extended .NET API uses native libraries in runtime, which depend on Windows platform:

Name Platform
SoftFX.Extended.dll Any CPU
SoftFX.Api.Sal.x64.dll x64
SoftFX.Sal.x86.dll x86
SoftFX.Core.x64.dll x64
SoftFX.Core.x86.dll x86
SoftFX.FixProvider.x64.dll x64
SoftFX.FixProvider.x86.dll x86
SoftFX.Native.x64.dll x64
SoftFX.Native.x86.dll x86
libeay32.dll x86
libeay64.dll x64
ssleay32.dll x86
ssleay64.dll x64

When SoftFX API is used at the first time FRE looks for the native libraries. You can specify location of native libraries by Library class using.

 In this case FRE tries to used native libraries from specified location only. If native libraries path was not specified then FDK looks for

  1. working directory of the current process;
  2. all directories of PATH environment variable;
  3. directory of the current process executable file;
  4. directory of the SoftFX.Extended.dll file.

If native libraries have not been found then FRE tries to load SoftFX.Extended.Native.Net.dll then extracts native libraries to %APPDATA%\SoftFX\FDK\<Library.Id>.
